BookKeeper - Library Management System

BookKeeper was designed for the management of large scale libraries from the ground up.

Run Instructions

  1. Copy all files to the computer.
  2. Open the BookKeeper directory.
  3. Run BookKeeper.exe
  4. Load Sample Data from File Menu (File > Load Sample Data)

Source Code

Location
You can view the source code in the following locations:
  • View the source code in the 'source' folder found in the root directory of the drive
  • View the source code on GitHub
Source Structure

The structure of the code reflects MVC organization:

  • The view model is stored in source\resources\library\fx
  • The controller layer is stored in source\java\library\fx
  • The model layer is stored in source\java\library\data

Usage

BookKeeper requires Java 8. If Java 8 is not installed, the program will use a bundled JVM to execute.

Features

  • Quickly editable patron information and catalogue information
  • Allows filtering and sorting in any table view
  • Includes customization of patron types
  • Generates reports in a matter of seconds
  • Allows librarians to override the system using their judgement
  • Cross Platform and can run on Windows, Linux, and MacOS
  • Includes fully-featured interactive documentation featuring images and videos
  • Includes an automatic data backup feature.

Planning

Design

BookKeeper uses a view, model, and controller system. The program uses JavaFX FXML files for its view layer. The controller layer is implemented through layers. The view is completely isolated from the models. The model layer is represented throught basic Java Objects. The model layer implements few functionalaties, as those are in the controller layer. This organization allows for losely coupled code and increases the efficiency of updating the program.

Database
Vertabelo was used to aid the design of the database

The layout of the database is designed to encourage modularity, e.g. changes in the patron table should be reflected in the book table automatically, without knowledge of the specific patron entry.
